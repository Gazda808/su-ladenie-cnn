Ladenie neurónovej siete CNN

Cieľ zadania

Cieľom tohto zadania je preskúmať vplyv jednotlivých hyperparametrov a architektonických rozhodnutí na kvalitu konvolučnej neurónovej siete (CNN). Implementovaný framework v PyTorchi umožňuje spustiť veľké množstvo experimentov ablačnou metódou – v každej skupine experimentov sa mení iba jeden parameter pri zachovaní ostatných.

Metodológia

Experimenty sú rozdelené do siedmich kategórií, pričom každá kategória izoluje vplyv jedného parametra:

  • Topológia – počet a šírka konvolučných blokov (od 1 do 3 blokov, 16 až 128 filtrov)
  • Optimalizátor – SGD, Adam, AdamW, RMSprop
  • Rýchlosť učenia – 0.0001, 0.001, 0.01, 0.1
  • Aktivačná funkcia – ReLU, LeakyReLU, ELU, GELU
  • Regularizácia – Dropout (0.25, 0.5), Batch Normalization, weight decay, kombinácie
  • Veľkosť jadra – 3×3, 5×5, 7×7
  • Veľkosť dávky – 32, 64, 128, 256

Celkovo bolo natrénovaných 34 experimentov na troch dátových sadách: MNIST, Fashion-MNIST a CIFAR-10.

Architektúra modelu

Použitá je flexibilná trieda ConfigurableCNN (src/models.py) postavená na nasledujúcom vzore:

Vstup → [Conv2d → BN? → Aktivácia → Pool → Dropout?] × N
       → Flatten → Linear → Aktivácia → Dropout? → Linear → Logit

Každý konvolučný blok obsahuje: konvolúciu s padding = kernel//2 (zachovanie priestorových rozmerov pred poolingom), volitelnú batch normalizáciu, aktivačnú funkciu a MaxPool2d/AvgPool2d s krokom 2. Klasifikačná hlava je plne prepojená vrstva s voliteľným dropoutom.

Mierky efektívnosti (hodnotenie)

Pre každý experiment sa zaznamenávajú a zobrazujú nasledujúce metriky:

  • Accuracy – podiel správne klasifikovaných vzoriek na testovacej sade
  • Cross-entropy loss – stratová funkcia, sledovaná na trénovacej aj testovacej sade
  • Macro Precision – priemer presnosti cez všetky triedy (bez váhy počtom vzoriek)
  • Macro Recall – priemer zachytenia cez všetky triedy
  • Macro F1-score – harmonický priemer precision a recall
  • Matica zámen – vizualizácia chybných klasifikácií pre každý pár tried
  • Per-class accuracy a F1 – identifikácia problematických tried
  • Krivky učenia – loss a accuracy po každej epoche (tréning aj validácia)
  • Počet parametrov a čas trénovania – efektívnosť modelu

Fungovanie programu

Systém pozostáva z dvoch vrstiev:

Backend (Python / PyTorch) — modul src/experiments.py definuje všetky experimenty deklaratívne (každý ako odchýlka od baselinovej konfigurácie). Skript src/train.py ich iteratívne spúšťa: načíta dáta cez src/datasets.py, zostrojí model (src/models.py), trénuje ho cez src/utils.py a na konci uloží kompletné výsledky (metriky, históriu, maticu zámen, ukážkové predikcie) do results/results.json a web/results.js.

Frontend (HTML/CSS/JS) — súbor web/app.js načíta predgenerovaný results.js a poskytne päť interaktívnych pohľadov: Prehľad s filtrovateľnou tabuľkou a stĺpcovým grafom, Porovnanie s krivkami učenia pre až 6 experimentov naraz, Detail s maticou zámen a ukážkovými predikciami, Konfigurátor na odhad výkonu novej konfigurácie a O zadaní.

Štruktúra projektu

projekt/
├── src/
│   ├── __init__.py
│   ├── datasets.py     # načítanie MNIST / Fashion-MNIST / CIFAR-10
│   ├── experiments.py  # definícia 34 experimentov
│   ├── models.py       # ConfigurableCNN
│   ├── train.py        # hlavný trénovací skript
│   └── utils.py        # optimalizátory, evaluácia, metriky
├── web/
│   ├── index.html      # SPA rozhranie
│   ├── app.js          # interaktívna logika
│   ├── style.css       # dizajn
│   └── results.js      # auto-generované výsledky
└── results/
    └── results.json    # surové výsledky experimentov

Implementácia a technologický stack

  • PyTorch 2.x – definícia a tréning CNN modelov
  • torchvision – načítanie a augmentácia dátových sád
  • NumPy – výpočet metrík (matica zámen, F1, precision, recall)
  • HTML5 / CSS3 / Vanilla JS – interaktívna SPA bez závislostí na frameworku
  • Chart.js 4 – krivky učenia, stĺpcové a skupinové grafy

Ako spustiť tréning

# Rýchle demo (menšia podmnožina dát, 2 epochy):
python -m src.train --quick

# Plný tréning na celých dátach (všetky experimenty):
python -m src.train

# Iba experimenty z jednej skupiny (napr. optimalizátory):
python -m src.train --filter opt_ --merge

# Vlastný počet epoch pre všetky experimenty:
python -m src.train --epochs 10 --merge

Po dokončení tréningu sa výsledky automaticky uložia do results/results.json a do web/results.js. Stačí znova načítať túto stránku — žiadny server nie je potrebný.
